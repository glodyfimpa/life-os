# life-os Plugin

## Architecture
- Plugin files: `.claude-plugin/plugin.json`, `commands/`, `skills/`, `skills/*/references/`
- User config: `.claude/life-os.local.md` (YAML frontmatter + markdown body, generated by `/setup`)
- Tool-agnostic: every read op falls back to "ask user", every write op falls back to "present in chat"
- Config fields `task_tool`, `calendar_tool`, `notes_tool`, `email_tool` control which MCP tools are used

## Conventions
- Skills reference config values, never hardcoded database IDs or field names
- `~~category` placeholders in CONNECTORS.md for tool-agnostic references
- Three modes: Full integration, Partial, Chat-only

## Skills Source
- Skills originate from `glodyfimpa/claude-skills` repo (source of truth)
- Sync via `./scripts/sync-skills.sh [path-to-claude-skills]`
- Skills are committed to life-os for plugin distribution (plugins are atomic, external refs don't survive install)
- Edit skills in claude-skills first, then sync to life-os

## Repos
- `life-os` — generic shareable plugin (GitHub: glodyfimpa/life-os)
- `claude-skills` — standalone skills repo (GitHub: glodyfimpa/claude-skills)
- `glody-life-os` — personal instance with local config
